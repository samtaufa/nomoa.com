Electronic Archiving

The reams of Gestetner Stencil papers are a testament to how much reprints are part of school systems.

The concept of Electronic Archiving is to archive electronic documents for re-use, distribution. The design will have to factor what are existing (approachable) technologies and the mechanism to installing and ensuring use.

Discussions on this proposal will need to review what are the basic needs for consistency in document creation and archiving methodologies.

Issues:

  • storage hierarchy / storage mechanism
  • storage format
  • security policies
  • backup policies
  • naming conventions

Storage Hierarchy

The hierarchy of document storage are categorised by intended audience while keywords (standardised by the design group) can be included in documents to maximise search / retrieval.

Storage Location: //storage/examinations/english/form5/[midyear|finals|topical]

contains documents relevant to the specified topic/course/level/section. Each document placed into this location can also include a "Keywords" section where related "section" information can be entered. Microsoft Word (for example) has a document properties component of the document where 'keywords' can be entered.

Files stored on Microsoft Windows NTFS 5.0 each have additional properties maintained by the File System that includes: Summary Data (including keywords)

Storage Format

Although it would seem appropriate to use a database, the use of an extra layer of work to store and retrieve the documents is likely to be an impediment to the use of the system. The quickest way to useage (as opposed to implementation) is to use a monitored hierarchical file system and store the documents in their original format.

Allow addition (file creation) to the system, but control deletion and modification. Staff can then check-out a document, make modifications and resubmit to the archive, but they cannot accidently wipe an existing document.

A simplified search utility can be developed to search everything or by specified keywords/categories. The utility can list the available documents, and attempt to launch them if the application exists on the client.

Security Policies

The archives will require a number of access privileges, but at minimum should have a public access policy to maximise the useage of the data store.

For example, once examination papers have been sat, all students should have read access to the papers.

Backup Policies

This is less likely to be pushed to conformity but a discussion should outline for participating schools a safe sequence of data backup procedures.

Considering the potential value of the system, it will be important to improve system integrity and up-time.

Naming Convention

As many documents of a similar nature will be created a standard naming convention should increase the useability of the system. (An area where creativity is likely to be counter-productive)
